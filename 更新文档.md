# Hand Particle System 更新文档

## 2026-05-24 GPU 粒子路径、手势指令、图片/Logo 和 GLB 点云导入

### 本次更新概览

- 重构图片/Logo 转粒子采样：放弃均匀步长采样，改为灰度化、3x3 高斯模糊、Sobel 梯度和边缘优先的加权自适应采样，输出点数直接贴近当前画质粒子数。
- 普通图片模式会把至少 80% 的采样分配给梯度排名前 20% 的强边缘区域，弱纹理内部按梯度和轮廓增强参数降采样，改善轮廓模糊和采样不均问题。
- 新增二值 Logo 模式：使用 Otsu 自适应阈值提取前景，只采样白色区域，并用形态学邻域边缘补偿提高 Logo 边界密度；内部采样比例可在 20%-50% 之间调节。
- 新增图片颜色映射选项：原色、灰度、单色三种模式，并通过 `aParticleColor` 顶点颜色直接把 RGBA 写入 shader；全局透明度会乘到粒子 alpha 上。
- 新增图片采样 UI 控件：轮廓增强、颜色模式、单色拾色器、透明度、二值 Logo 模式和内部采样比例。
- 为控制 WebGL attribute 数量，将随机值和颜色混合值打包为 `aParticleParams`，新增顶点颜色后仍保持移动端可用的 attribute 数量。
- 新增 GPU 顶点 shader 粒子模拟路径：粒子目标点、方向、半径、亮度、类型等写入 GPU attribute，每帧扩散、音乐扰动、烟花爆开、转场散射和指向偏移主要由 shader 计算，减少 CPU 对 150k-300k 粒子的逐粒子位置更新。
- 对 WebGL attribute 做打包优化，把半径、类型、基础亮度合并为 `aTargetMeta`，并复用随机相位推导部分扰动参数，避免移动端出现 attribute 数量超限。
- 增强手势指令系统：保留张开爆散、收拢聚回；新增食指指向，让模型朝手指方向移动；所有“手势切换模型”行为已暂时关闭。
- 新增图片/Logo 转粒子：上传图片后读取像素透明度、亮度和颜色，过滤透明/浅背景区域并生成可被手势和音乐驱动的粒子点云。
- 新增 GLB 模型粒子化：通过 `GLTFLoader` 读取 `.glb`，遍历网格三角面并按面积采样模型表面，归一化后生成真正有深度的 3D 粒子点云。
- UI 模型区新增“图片”和“3D”入口，控制面板新增“导入”区域，用于选择图片/Logo 和 GLB 文件。

### 修改文件

- `src/particles.js`：加入 GPU attribute 驱动的顶点 shader 位置模拟，新增 `aParticleColor` 顶点颜色和 attribute 打包，移除主路径中的每帧 CPU 逐粒子位置循环，并新增图片/模型点云目标入口。
- `src/shapes.js`：新增通用点云目标写入函数，供图片和 GLB 采样结果复用，并把点云 RGBA 写入顶点颜色。
- `src/gestures.js`：新增双手比心、食指指向识别；OK/比心识别保留但不再触发模型切换。
- `src/main.js`：接入手势指令、图片边缘自适应采样、Logo 二值采样、GLB 表面采样和指向位移。
- `index.html` / `src/ui.js` / `src/styles.css`：新增图片、3D 模型按钮、导入控件和图片采样参数控件。
- `README.md` / `更新文档.md`：同步高级功能说明。

### 本次更新概览
