# Hand Particle System

实时交互 3D 粒子系统：Three.js 渲染粒子，MediaPipe Hands 检测双手张合，UI 支持模型切换、颜色选择和全屏。

## 运行

```bash
npm install
npm run dev
```

浏览器打开：

```text
http://localhost:5173
```

建议使用 Chrome 或 Edge 打开，并在浏览器弹窗里允许摄像头权限。不要直接双击打开 `index.html`，也不要用 `dist/index.html` 的本地文件路径打开。

## 手势使用

1. 打开页面后，先看左侧面板底部的视频预览，确认能看到自己的摄像头画面。
2. 把完整手掌放进预览框，手掌正对摄像头，手腕和五根手指尽量都入镜。
3. 单手张开时，爱心会扩散、变亮；单手收拢成拳时，爱心会聚回模型。
4. 保持单手握拳并旋转拳头时，会进入拳头视角控制：拳头旋转控制水平环绕，拳头上下移动控制俯仰。
5. 双手同时入镜时，双手距离越远，整体扩散越明显；双手靠近会收拢。
6. 鼠标拖动画面可以旋转视角，滚轮缩放；按住 Shift 拖动可以不用手势也扰动粒子。

## 识别诊断

左侧状态文字下面有一行诊断信息：

- `模型已加载，正在请求摄像头`：模型正常，等待浏览器摄像头授权。
- `摄像头未就绪`：检查浏览器地址栏的摄像头权限，或确认摄像头没有被其他软件占用。
- `摄像头有画面，但模型还没有返回结果`：模型循环还没开始返回结果，刷新页面通常能恢复。
- `尚未识别到手`：模型在运行，但画面里没有检测到完整手掌；把手放远一点，让整只手入镜。
- `识别到 1 只手` / `识别到 2 只手`：识别成功，后面的百分比就是当前控制粒子的张合值。

## 常见问题

- 必须用 `http://localhost:5173` 打开。摄像头 API 在普通局域网 IP 的 `http://` 页面上可能被浏览器拦截；如果要在手机或另一台电脑上访问，需要配置 HTTPS。
- 摄像头画面太暗、逆光、手离镜头太近、只露出几根手指，都会导致 MediaPipe 识别不到。
- 第一次加载会读取 `public/mediapipe/hands` 下的 wasm 和模型文件，不要删除这些文件。
- 如果浏览器地址栏显示摄像头被禁用，重新允许权限后刷新页面。
- `npm run verify` 使用的是假摄像头，只能验证页面和资源是否正常，不能代表真实手势识别效果。

## 依赖

- `three`：3D 场景、粒子和后期 Bloom。
- `@mediapipe/hands`：本地手部关键点检测。
- `vite`：开发服务器和构建。
- `playwright`：仅用于本地截图验证。

## 验证

```bash
npm run build
npm run verify
```

`npm run verify` 会生成 `verification-desktop.png` 和 `verification-mobile.png`。
